Скрипт пользования гитом

  • Открываем консоль. выбираем диск, на котором наш проект E: . Выбираем папку при помощи команды cd имяпапки, содержание тома можно посмотреть при помощи команды dir
  • Зайдя в папку проекта при помощи команд проверили git config --global user name (мое имя должно совпадать с именем на GitHub) и git config --global user.email (то же самое). Если не совпадает, git config --global user.name "имясGitHub" (то же самое касается и user.email)
  • Заходим на GitHub, авторизируемся и заводим новый репозиторий
  • В нашей консоли инициализируем Git в папке проекта git init
  • Для получения статуса проекта в гите прописываем git status
  • Для изменения одного файла прописываем git add имяфайла Для изменения нескольких файлов проекта прописываем git add .
  • Переключились на GitHub. Новый репозиторий предлагает следующее:
    • echo "# git-project" >> README.md - создает ридми через консоль
    • git init - инициирует Git в папке, в которой находимся
    • git add README.md - добавляет ридми в репозиторий проекта
    • git commit -m "first commit" - делает первый комит на компе
    • git branch -M main - переминование главной ветки master в main (если нужно)
    • git remote add origin https://github.com/Vyxopylu/git-project.git - привязка нашего репозитория на GitHub к локальному. Копируем, вставляем
    • git push -u origin main - для того, чтобы залить то, что у нас на локальном гите на GitHub юзаем это

В дальнейшем работа с репозиторием осуществляется по следующей схеме:

  • Запускаем проект
  • Заходим через консоль в папку
  • Работаем с проектом
  • Доработав до определенной логической точки, сохраняем изменения при помощи git add
  • Из изменений собираем коммит git commit -m "нашесообщение"
  • При помощи git push заливаем коммит (или несколько собравшихся коммитов на GitHub

Для того, чтобы Git игнорил изменения в каком либо файле, создаем в корне проекта файл .gitignore, где прописываем имя файла, который мы хотим игнорировать. Не забываем применить на .gitignore команду git add

Получаем ссылку на свой проект:

  • На странице проекта GitHub заходим в settings и переключаемся на вкладку pages(справа)
  • В пункте Branch выбираем ветку из которой хотим получить ссылку на страницу проекта, жмем save
  • Ждем какое-то время, обновляя страницу. Минут через 2-5 появится ссылка. Пользкемся
  • Для большего удобства закидываем ссылку в файл README.md. Данный тип файлов - файл со спецразметкой. Ссылку закидываем в формате (комментарий куда вообще ведет ссылка)[ссылка на проект]
